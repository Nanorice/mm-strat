# M01 Model (Return Regressor) - Module Reference

## 1. Overview
*   **Responsibility:** The M01 module is responsible for training and executing the **SEPA Return Regressor**. It predicts the expected percentage return (specifically a `log_space` target) for trade candidates generated by the SEPA strategy scanner. It ranks candidates to prioritize those with the highest potential favorable excursion.
*   **Key Dependencies:**
    *   **External:** `xgboost` (Model), `pandas`, `numpy`, `joblib`.
    *   **Internal:** `src.pipeline` (BaseTrainer, DataPipeline), `src.feature_config` (Feature sets), `src.evaluation` (Evaluator, TargetEngineer), `src.feature_preprocessor`.

## 2. File Structure

| File Path | Role / Purpose |
| :--- | :--- |
| `model_runner.py` | **Entry Point**. CLI for executing pipelines (`scan`, `features`, `train`) and workflows. |
| `src/pipeline/m01_trainer.py` | **Core Logic**. Contains the `M01Trainer` class, defining the XGBoost model, target calculation (`log_space`), and survivor model logic. |
| `src/pipeline/m01_workflow.py` | **Orchestrator**. Automates the end-to-end flow: Load Data → Auto-EDA → Feature Selection → Train → Report. |
| `src/pipeline/data_pipeline.py` | **Data Factory**. Generates the datasets (D1, D2, D2R) required by M01. |
| `src/feature_config.py` | **Configuration**. Single source of truth for `M01_FEATURES` and other feature sets. |
| `src/feature_preprocessor.py` | **Preprocessing**. Handles log-transformations and winsorization based on feature names. |

## 3. Data Schemas

### D1: Trade Candidates (Scan Output)
*Intermediate storage: `data/ml/d1.parquet`*

| Column | Type | Description |
| :--- | :--- | :--- |
| `date` | datetime | Entry date of the trade candidate. |
| `ticker` | string | Stock ticker symbol. |
| `label` | int/bool | Basic success label (Hit 15% threshold? 1/0). |
| `return_pct` | float | Actual return of the trade (raw). |
| `days_held` | int | Duration of the trade. |
| `exit_reason` | string | Reason for exit (e.g., 'Stop Loss', 'Target', 'Time'). |

### D2: Enriched Features
*Intermediate storage: `data/ml/d2.parquet`*
Includes all D1 columns plus **M01 Features**. Key features from `M01_FEATURES`:

| Feature Category | Examples |
| :--- | :--- |
| **Technical (Log)** | `log_nATR`, `log_Price_vs_SMA_200`, `log_Dist_From_52W_Low`, `log_RS_Delta` |
| **Alpha Factors** | `alpha011`, `alpha013`, `alpha060`, `alpha101` |
| **Fundamentals** | `operating_margin`, `log_current_ratio`, `eps_stability_score` |
| **Velocity** | `rs_velocity`, `log_volume_velocity` |
| **Regime (M03)** | `m03_score`, `m03_regime_cat` (if `include_m03=True`) |

### M01 Target Schema (Calculated in Trainer)

| Column | Formula / Logic | Description |
| :--- | :--- | :--- |
| `target` | `sign(MFE) * log(1 + abs(MFE))` | **Primary Target**. Log-space compression of Max Favorable Excursion (MFE) to smooth tails. |
| `y_max` | `MFE` (if Survivor) else `MAE` | **Survivor Target**. Used if `--survivor` is enabled. |
| `is_survivor`| `MAE > structural_stop` | **Survivor Flag**. True if trade did not hit the structural stop. |

## 4. Implementation Rules

### Modeling & Logic
*   **Target Definition**: The default target is `log_space` (IC=0.338 winner).
    *   Formula: $y = \text{sign}(MFE) \times \ln(1 + |MFE|)$
*   **Survivor Model Strategy**:
    *   **Structural Stop**: Hardcoded logic uses `stop_multiplier`.
    *   Rule: `structural_stop = -1 * stop_multiplier * nATR`.
    *   Barrier: Trades hitting this stop (MAE <= structural_stop) are marked as `is_survivor=False`.
*   **Success Threshold**:
    *   Scan Step: Default `15.0%` return needed to trigger a generic success label (though M01 is a regressor).
*   **XGBoost Hyperparameters** (Defaults in `get_model_params`):
    *   `n_estimators`: **300**
    *   `learning_rate`: **0.03**
    *   `max_depth`: **4**
    *   `subsample`: **0.8**, `colsample_bytree`: **0.8**
    *   `reg_alpha` (L1): **5.0**, `reg_lambda` (L2): **3.0**

### Preprocessing Rules
*   **Log Transformation**: Applied to any feature starting with `log_`.
    *   Logic: `sign(x) * log(1 + |x|)` (handles negatives).
*   **Winsorization**: Configurable in preprocessor, clips extreme outliers.

## 5. Public Interface

### `User-Facing CLI (model_runner.py)`
*   `python model_runner.py m01 --steps scan features train`
*   `python model_runner.py workflow --start [DATE] --end [DATE]`

### `class M01Trainer` (`src/pipeline/m01_trainer.py`)
Responsible for training the model.

| Method | Key Arguments | Return |
| :--- | :--- | :--- |
| `train` | `data` (D2), `tune` (bool), `target` ('log_space'), `survivor_model` (bool) | `(model, metrics_df)` |
| `enrich_with_survivor_labels` | `data`, `stop_multiplier` (default 2.0) | DataFrame with `y_max`, `is_survivor` added. |
| `generate_report` | `model`, `metrics_df`, `start_date`, `end_date` | Path to markdown report. |

### `class DataPipeline` (`src/pipeline/data_pipeline.py`)
Responsible for generating the data artifacts.

| Method | Role | Output |
| :--- | :--- | :--- |
| `scan(start, end, threshold)` | Simulates SEPA trades. | `d1.parquet` |
| `features(d1, include_m03)` | Enriches trades with features. | `d2.parquet` |
| `hydrate(d1, horizon)` | Fetches multi-day price trajectories. | `d2r_*.parquet` |

### `class M01Workflow` (`src/pipeline/m01_workflow.py`)
Automated factory for model development.

| Method | Key Arguments | Role |
| :--- | :--- | :--- |
| `run` | `steps=['load', 'eda', 'select', 'train', 'report']` | Executes the full pipeline, including feature selection (EDA). |
