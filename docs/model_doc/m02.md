## 0. TL;DR / Status

- **Status:** `PROTOTYPE — PARKED as a strategy` (2026-07-04). Event prediction validated OOS,
  but the Phase-0 signal-quality gate returned **NO-GO for a trade build**: top-50 forward
  returns ≈ universe mean (+12bps/21d pre-cost). The score predicts the *watchlist event*,
  not returns. See §6 and
  [m02_signal_quality_report.md](../session_logs/sprint_13/m02_signal_quality_report.md).
- **What it does:** ranks the full ticker universe each day by *how close each name is to
  igniting* — its next SEPA-watchlist breakout. A concentrated shortlist ("watch these ~50").
- **Champion:** `m02_breakout` (XGBoost regressor, target = `breakout_proximity`).
  Latest run `models/m02_breakout/20260629_203134/`.
- **Headline metric:** **Precision@50 ≈ 50%** out-of-sample (2021–2026, 5 anchored WF folds)
  vs a **13.66% base rate** → **~3.6× edge**. Rank IC **+0.33 to +0.39** (mean +0.37).

## 1. Problem & Purpose

M01 ranks candidates *once they've already surfaced* as SEPA trade setups (sparse, event-driven).
Nothing in the pipeline answers the earlier question: **"which names in the whole universe are
about to ignite?"** — a pre-breakout early-warning ranker over *all* tickers, every day.

That's M02's single job: turn the sparse, binary "entered the watchlist" event into a dense,
trackable gradient so we can watch the ~50 names most likely to break out next, before they trip
M01's scanner.

## 2. Theory / Hypothesis

A structural breakout (VCP-style base completion, supply/demand tightening) leaves a
*measurable pre-signature* in price/volume velocity and momentum in the days–weeks before the
event. If that signature exists, a model trained on the dense population should rank names by
time-to-ignition with better-than-random precision — **even though the ignition event itself is
rare (~14% within 60d).** The 50% Precision@50 result (§5) is the evidence that the signature is real.

## 3. Specification

| Aspect | Value |
| :--- | :--- |
| **Algorithm** | XGBoost regressor, `reg:squarederror`, `hist` tree method |
| **Target** | `breakout_proximity = exp(-k · days_to_next_watchlist_entry)`, k=0.1, 0 beyond 60d horizon |
| **Feature set** | `fs_m01_prototype` (dense candidate-population features) via `t3_training_cache` |
| **Universe** | Dense — all tickers, all days (~9.3M rows, 2021–2026 test span) |
| **Key hyperparams** | `max_depth=6`, `eta=0.05`, `subsample=0.8`, `colsample_bytree=0.8`, `min_child_weight=20`, `num_boost_round=300` |
| **CV geometry** | Anchored walk-forward, 1Y step, **60-day embargo** (= horizon) to block lookahead |

**Target construction** (`scripts/build_breakout_targets.py`): join `price_data` against
`sepa_watchlist`, compute calendar days to each ticker's *next* watchlist entry, apply the
exponential decay. Ignites tomorrow → ~1.0; in 30d → ~0.05; no ignition in 60d → exactly 0.0.

## 6. Version History / Journey

Ignition classifier only. (Pre-ignition uses of the "M02" name are in §9, not here.)

- **Sprint 13 — conceived and validated.** Reframed breakout prediction onto the *structural
  event itself*: continuous proximity to the next watchlist entry, trained on `fs_m01_prototype`
  dense features. **Result: 50% P@50, ~3.6× edge** out-of-sample. See
  [m02_breakout_model.md](../research/m02_breakout_model.md) and
  [sprint13_summary.md](../session_logs/sprint_13/sprint13_summary.md).
- **2026-07-04 — finalized (G1/G2), then trade thesis falsified (Phase-0 gate).** Final
  all-period fit + frozen metadata shipped (`models/m02_breakout/final_20260704_175544/`).
  The signal-quality gate (`scripts/analyze_m02_signal_quality.py`, all scores OOS via the
  WF fold models + final-model post-train tail) confirmed the event signal — 91.7% of
  watchlist entries had a prior top-decile signal, median 42d lead; ignition rate monotone
  by decile (2.2%→41.6%) — but **falsified the return claim**: forward returns peak at
  decile 7 and fall into decile 10, and the daily top-50 earns ≈ the universe mean
  (+12bps/21d excess pre-cost, negative 2022–2025). The apparent head-start return
  (+4.16% vs +0.56%) was selection bias (conditioning on eventual igniters). Structural
  read: proximity peaks where the pre-breakout move is already spent — **predicting the
  scanner event ≠ predicting returns**. Strategy build (Goal A3 Job 1) declined; model
  parked as event-predictor only. Full numbers:
  [m02_signal_quality_report.md](../session_logs/sprint_13/m02_signal_quality_report.md).

## 7. Usage

```bash
# Build dense proximity targets (joins price_data × sepa_watchlist → m02_breakout_targets)
.venv/Scripts/python.exe scripts/build_breakout_targets.py

# Train + anchored-WF eval (checkpointed per fold; --smoke for a 5-ticker path test)
.venv/Scripts/python.exe scripts/train_breakout_model.py --smoke        # smoke first
.venv/Scripts/python.exe scripts/train_breakout_model.py                 # full run
```

- Output: `models/m02_breakout/<run_tag>/` — per-fold `*_model.json`, `fold_NN.json`
  checkpoints, `summary.json`.
- **Not yet in prod.** To go live it must be wired into `daily_scanner.py` and surfaced in the
  dashboard for forward, out-of-sample monitoring (see §8).

## 8. Roadmap / Open Questions

### 8a. Prototype → production gap (assessed 2026-07-04)

> Implementation plan: [m02_prod_gap_plan.md](../session_logs/sprint_13/m02_prod_gap_plan.md)
> (phased, gated on the Phase-0 arena verdict). G1+G2 closed 2026-07-04 —
> `models/m02_breakout/final_20260704_175544/` (model.json + metadata.json).

m02_breakout is validated but **prototype-grade**. Measured against the M01 artifact standard
(`models/m01_binary/v1/`), it is missing most of the production scaffold. Gaps, in order:

| # | Gap | M01 has | m02_breakout has | Why it matters |
|---|---|---|---|---|
| **G1** | **Final all-period model** | single `model.json` | 5 per-fold `fold_NN_model.json` only | WF folds are *validation*, not a deployable artifact. Need one booster fit on all data → one clean scoring pass, no fold-routing. **(Goal A3 §0 `--final` flag.)** |
| **G2** | **Frozen metadata** | `metadata.json` (feature list, label def) | none | scoring/serving needs the exact feature vocab pinned, not re-derived from `model_feature_sets` at call time. |
| **G3** | **Score materialization** | `daily_predictions` rows + dashboard | nothing materialized | can't monitor live P@50 or feed the backtester/scanner without a persisted score panel. |
| **G4** | **Calibration** | `calibrator.joblib` (isotonic) | raw regressor output | `breakout_proximity` is an uncalibrated decay score. For a *threshold* strategy ("enter above X") the score must mean something stable across time — needs a calibration map or an explicit "rank-only, never threshold on raw value" contract. |
| **G5** | **Pre-train audit / reference snapshot** | `pretrain_audit.html`, `reference_snapshot.json` | none | leakage/feature-drift guards + a frozen reference distribution for live drift detection. |
| **G6** | **Backtest** | `backtests/`, `wf_backtest/` | none | never run through the strategy engine — the whole point of Goal A3. |
| **G7** | **Model card** | `evaluation/`, `model_card.html` | `summary.json` only | no shipped, reviewable evaluation surface. |

**Priority for "next level":** G1 → G3 → G6 (get a deployable model, materialize its scores,
backtest it) are the load-bearing three — they turn it from "a validated notebook result" into
"a model that participates in the arena and can go live." G2/G4 ride along with G1. G5/G7 are the
governance layer, do them when promoting past prototype (they gate the PROTOTYPE→PROD flip in G3
below). **Do NOT** treat G4 (calibration) as optional if any strategy thresholds on the raw score
— until then, the honest contract is **rank-only**.

### 8b. Roadmap after the Phase-0 NO-GO (updated 2026-07-04)

The prod-gap plan's Phases 2–4 (nightly scoring, governance, monitoring) **do not run** on a
strategy justification — the gate failed (§6). What remains open:

1. **Ops-tool decision (open, user call)** — the event signal is real and broad; the only
   defensible prod use is an *attention list* ("likely to trip the M01 scanner within weeks"),
   with an explicit **no-return-claim** contract. Smaller build than the parked plan's Phase 2.
2. **Research reframe (if revisited)** — the falsification is specific: *proximity to the event*
   is the wrong target for returns because it peaks after the move. A return-bearing reframe
   would target *forward return conditional on proximity* (e.g., decile-7 shape) or move the
   horizon earlier — new model, new doc section, not a tweak.
3. **Retired ideas** — target stratification / k-horizon tuning as paths to a trade: sharpening
   event prediction sharpens the wrong thing. Exit-engineering (E2/E3) on a ~0-mean edge: overfit
   bait, declined.
