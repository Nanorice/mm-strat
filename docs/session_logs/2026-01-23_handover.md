## 💡 Key Learnings & Design Decisions

### 1. **Ignition vs Quality are Different Problems**
- **M01 Question:** "Will this trade return 15%+ eventually?" (Quality/Direction)
- **M01_3BAR Question:** "Will this trade EXPLODE within 30 days?" (Velocity/Speed)
- **Insight:** Same features can't answer both - need specialized velocity features

### 2. **Phase 1 Ignition Score = 2.17 is STRONG**
- Optimized barriers separate TP (25.9% return) from Time (6.2% return) by 2.17 std devs
- This proves the LABEL quality is excellent
- If model edge is weak, it's a FEATURE problem, not a label problem

### 3. **nATR Dominance is a Red Flag**
- nATR at #1 (11.9%) means model is trading volatility regimes
- "High volatility stocks hit barriers faster" is physics, not alpha
- Need to remove nATR from feature set to force model to find true edge

### 4. **Velocity Features ARE Working (Partially)**
- 3/8 velocity features in Top 20 proves signal exists
- `consolidation_duration` (#15) validates "coil length matters"
- `volume_acceleration` (#17) proves "buying surges are predictive"
- `alpha046` (#13) shows slope change detection has value

### 5. **Feature Surgery > Feature Addition**
- Adding velocity features helped, but not enough
- Model needs FEWER features (remove escape hatches)
- Quality: "Is it good?" ≠ Velocity: "Is it fast?"

## 💡 Key Design Decisions (Session 2)

### 1. **EDA Utility Module Pattern**
**Decision:** Create `src/eda_utils.py` instead of inline notebook functions

**Rationale:**
- **Reusability:** Same functions for M01, M01_3BAR, future models
- **Testability:** Can unit test analysis logic independently
- **Maintainability:** Bug fixes apply to all notebooks
- **Clean notebooks:** Focus on storytelling, not implementation

**Impact:** Notebook is 70% smaller, much more readable

### 2. **Programmatic Notebook Building**
**Decision:** Use Python script (`build_eda_notebook.py`) instead of NotebookEdit tool

**Rationale:**
- **Efficiency:** Generate 13 cells in seconds vs minutes
- **Consistency:** Guaranteed formatting and structure
- **Version control:** Notebook source code in .py file
- **Extendability:** Easy to add Sections 2-4 later

**Trade-off:** Less interactive during development, but faster overall

### 3. **Industry-Standard Metrics**
**Decision:** Implement E-Ratio, KS test, ECE, NPV (not just accuracy/AUC)

**Rationale:**
- **E-Ratio (MFE/MAE):** Industry standard for breakout strategy validation (benchmark: 3.0)
- **KS test:** Proper statistical test for feature discrimination
- **ECE (Expected Calibration Error):** Validate model confidence matches reality
- **NPV (Negative Predictive Value):** Prove low scores are "death sentences"

**Impact:** Professional-grade analysis, publication-ready
