## 💡 Context/Memory

### HY OAS proxy decision
`BAMLH0A0HYM2` (ICE BofA HY Index) is now FRED-restricted to last 3 years (ICE licensing, April 2026). The 5-factor prototype already solved this: use `WBAA - DGS10` (Moody's Baa yield minus 10Y Treasury) as a proxy. Same widening/tightening signal, available from 1986. This is baked into `risk_5_factor.py` — do not use `BAMLH0A0HYM2` directly for the 5F model.

### WBAA ffill before subtraction (critical)
WBAA is weekly. It must be reindexed to the SPX daily calendar and ffill'd **before** subtracting DGS10, otherwise the subtraction produces NaN on non-Friday days that survive the later ffill pass. This matches the prototype exactly and is correctly implemented.

### SPX ticker in price_data
`^GSPC` is stored with the caret prefix in `price_data` (DuckDB VARCHAR PK handles it). The 5F model queries `WHERE ticker = '^GSPC'` with an SPY fallback if missing.

### 5F vs M03 are additive (EDA confirmed)
From 2026-05-09 EDA: `M03>60 AND 5F≥0.75` combined gate gives IC=0.105 vs M03 alone (0.094) or 5F alone (0.097). Both models should be used together as regime gates in the strategy, not as substitutes. The 5F `target_exposure` >= 0.75 corresponds to `rolling_percentile` < 0.55 (moderate risk).

### Exposure band mapping
`rolling_percentile` maps to `target_exposure` as:
- 0.00–0.20 → 100% (low risk)
- 0.20–0.40 → 85%
- 0.40–0.55 → 75%
- 0.55–0.70 → 50%
- 0.70–0.85 → 35%
- 0.85–1.00 → 15% (high risk)
- Any veto (z ≥ 2.0) → 15% override

### t2_risk_scores warmup
With `ROLLING_WINDOW_PCT = 1260` (5yr) and data from 1990, scored rows start from ~1995. With `ROLLING_WINDOW_Z = 2555` (10yr), z-scores start from ~2000. The table will have NaN `target_exposure` for the warmup period — this is correct and expected.
