## 🚧 Critical Issues & Architectural Debt

### 1. **Barrier Parameter Confusion (FIXED but needs documentation)**
**Issue:** `horizon_days` (120d rehydration) was confused with `max_time` (30d barrier timeout)
- `horizon_days = 120`: How far forward we look to calculate features
- `max_time = 30`: When we give up waiting for TP/SL and exit as "Time"
- **Fixed:** Updated model report to show both separately (lines 1562-1563)
- **Action Needed:** Document this distinction in architecture docs

### 2. **Duplicate Model Versions Problem**
**Current State:** We now have TWO versions of M01_3BAR:
- `M01_3BAR` (baseline, 21 features)
- `M01_3BAR_V2` (velocity, 29 features)

**Problems:**
- Duplicate code paths in model_trainer.py
- Confusion about which to use
- Need to maintain both feature configs

**Solution (Next Refactor):**
- **Option A:** Merge into single M01_3BAR with ALL features, let XGBoost do feature selection
- **Option B:** Keep V2 only, deprecate baseline (since Phase 1 proved velocity features help)
- **Recommended:** Option B - just use V2 as the canonical M01_3BAR

### 3. **Hardcoded Barrier Parameters**
**Current State:** Barrier params are hardcoded in model_trainer.py (line 1890-1893)
- Changed from old (k_sl=2.0) to optimized (k_sl=1.0) manually
- No connection to Phase 1 optimization results

**Better Solution (Future):**

- Model is using fundamentals (`pe_ratio`, `operating_margin`) instead of velocity

**Root Cause:** Feature set is too diverse - model has "escape hatches" to avoid learning velocity patterns

## 💡 Key Learnings & Design Decisions

### 1. **Ignition vs Quality are Different Problems**
- **M01 Question:** "Will this trade return 15%+ eventually?" (Quality/Direction)
- **M01_3BAR Question:** "Will this trade EXPLODE within 30 days?" (Velocity/Speed)
- **Insight:** Same features can't answer both - need specialized velocity features

### 2. **Phase 1 Ignition Score = 2.17 is STRONG**
- Optimized barriers separate TP (25.9% return) from Time (6.2% return) by 2.17 std devs
- This proves the LABEL quality is excellent
- If model edge is weak, it's a FEATURE problem, not a label problem

### 3. **nATR Dominance is a Red Flag**
- nATR at #1 (11.9%) means model is trading volatility regimes
- "High volatility stocks hit barriers faster" is physics, not alpha
- Need to remove nATR from feature set to force model to find true edge

### 4. **Velocity Features ARE Working (Partially)**
- 3/8 velocity features in Top 20 proves signal exists
- `consolidation_duration` (#15) validates "coil length matters"
- `volume_acceleration` (#17) proves "buying surges are predictive"
- `alpha046` (#13) shows slope change detection has value

### 5. **Feature Surgery > Feature Addition**
- Adding velocity features helped, but not enough
- Model needs FEWER features (remove escape hatches)
- Quality: "Is it good?" ≠ Velocity: "Is it fast?"
