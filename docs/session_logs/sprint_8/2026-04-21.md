## 💡 Context/Memory
- **Why no alias for `trailing_10d_pct`**: The old column name baked in the 10-day window as if it were fixed. Now that `ranking_lookback_days` is configurable, "10d" in the name would mislead. Clean rename forced; stale files raise immediately rather than silently using the wrong value.
- **Why `prob_elite` is not threaded into `valid_candidates`**: `_process_entries` builds `valid_candidates` as `(ticker, score, trailing_pct, data)` — `data` is the Backtrader feed object needed for position sizing. `prob_elite` was already used as a filter in `get_candidates()`; the strategy doesn't need it downstream (ranking already done). Adding it would change downstream 4-tuple unpack without any benefit.
- **T3 vs d2 data gap**: `d2_training_cache` is the view `v_d2_hydrated` which pulls from `daily_features` + joins fundamentals; it has the full 149-col feature set. `t3_sepa_features` only has the 150 cols from `daily_features` itself (no fundamentals joined). This is why the prototype model scores differently on T3 — those fundamentals columns go to NaN.
- **Warmup default is 0**: Not 10. If you set `warmup_days=10` and `ranking_lookback_days=10`, the first 10 bars are filled with progressively smaller trailing-percentile cohorts but no entries happen — so day 11 is the first meaningful entry. The default of 0 preserves backward-compatibility (existing runs don't change behavior).
