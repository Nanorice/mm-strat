# Session Handover: 2026-02-20

## 🎯 Goals
1. **Dynamic Universe Backfill**: Implement an architecture to eliminate survivorship bias by storing ~4,500 US tickers × 20 years of OHLCV + shares data in immutable backfill tables completely isolated from the daily production pipeline.
2. **Historical Shares Tracking**: Track historical shares via `shares_history` to enable point-in-time accurate market cap and valuation ratio calculations for backtesting.

## ✅ Accomplished

### 1. Dynamic Universe Backfill Engine (`src/universe_backfill.py`)
- **`UniverseBackfillEngine` class**: Contains `discover_tickers()` (FMP screener/delisted), `add_tickers_from_csv()` (CSV fallback), `backfill_prices()` (yfinance batched OHLCV), `backfill_shares()` (8-worker ThreadPool yfinance), and `compute_universe_snapshots()` (SQL monthly membership).
- **Graceful execution & checkpoints**: Built-in resume capabilities directly querying DuckDB tables; no external state files. Data flushed every 500 tickers.
- **`scripts/run_universe_backfill.py`**: CLI orchestrator with flags for `--discover`, `--backfill-prices`, `--backfill-shares`, `--compute-snapshots`, `--add-csv`, `--status`, `--validate`, and `--all`.

### 2. Historical Shares Tracking (`src/shares_engine.py`)
- **`SharesEngine` class**: Fetches full history (`backfill()`) or incremental (`update()`) using parallel `yfinance` fetches. Added `--update-shares` and `--force` flags to `data_curator_duckdb.py`.
- **`shares_history` Table**: Added to schema (ticker, date, shares_outstanding). End-to-end tested with 5 tickers (AAPL, MSFT, NVDA, TSLA, GOOGL).
- **Integrations**: Integrated into `--update-all` phase 1 pipeline. Deleted prototype `check_shares.py`.

### 3. Pipeline & View Integrations (`src/view_manager.py`, `src/feature_pipeline.py`)
- **Combined Backfill Views**: Added `_create_v_price_combined()` and `_create_v_shares_combined()` in `view_manager.py` (UNION ALL anti-joins) with graceful fallbacks if backfill tables are missing.
- **Point-in-Time Features**: Updated `_create_v_d2_features()` to use ASOF-style joins (`MAX(date) WHERE date <= d1.date`) on `shares_history` with fallback to `cp.shares_outstanding` for backward compatibility.
- **Pipeline Parametrization**: Added `use_backfill: bool = False` flag to `FeaturePipeline` so production workflow has zero impact unless explicitly toggled on.

### 4. Schema Updates (4 New Tables)
- Supported by `migrate_to_duckdb.py` schema initialization:
  - `master_ticker_registry`: ~4,500 active + recent delistings
  - `price_data_backfill`: OHLCV for backfill tickers (immutable)
  - `shares_backfill`: Shares outstanding for backfill (immutable)
  - `universe_snapshots`: Monthly universe membership + filter flags
  - `shares_history`: PK on (ticker, date)

## 🚧 Work in Progress (Limitations & Edge Cases)

### FMP Delisted Limitation
- `/delisted-companies` endpoint only returns ~100 recent delistings (2026-01-26 onwards); lacks historical delistings.
- **Workaround/Decision**: Use `add_tickers_from_csv()` for CRSP/Sharadar integration later. Accepted limitation - backfill remains high-value for current 4,452 active stocks × 20 years.

### Untested Scenarios & Potential Issues
- **Full Universe Execution**: Never run end-to-end (2+ hours, API quota risk). Code is logically sound but full-scale execution untested.
- `yf.download()` edge cases: MultiIndex unpacking or missing tickers / sparse shares. Handled via COALESCE and individualized fallbacks but unverified on tail end.
- **DuckDB Performance**: 20M+ rows in UNION ALL with anti-join may be slow.

## 💡 Context/Memory
- **Architecture**: Immutability (`INSERT ... ON CONFLICT DO NOTHING`), checkpointing (DuckDB tables are the state), and non-invasive views (default to skipping backfills).
- **ASOF Join Pattern**: DuckDB lacks native ASOF, so we use `MAX(date) WHERE date <= d1.date`. Performs robustly for point-in-time accuracy.
- **FMP Screener**: `/company-screener` is the optimal endpoint containing full metadata compared to `/stock-list`.
- **Shares Design**: Kept `float_shares` in `company_profiles` as snapshot. Only `shares_outstanding` tracked historically due to tracking sparsity/yfinance quirks.
