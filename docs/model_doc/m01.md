## 1. Problem & Purpose

The SEPA scanner surfaces trade candidates (trend + breakout structure passes). On a busy day
it surfaces more than the book can hold. M01's job is the **ranking decision**: given today's
candidates, which are most likely to produce a large favorable move? It is a *selection* model —
it does not decide *whether* to be in the market (that's regime/timing), only *which names*
among those already qualified.

## 2. Theory / Hypothesis

Among names that already pass the SEPA structural filter, cross-sectional differences in
momentum, relative strength, volume dynamics, fundamentals, and regime context carry
information about *which* will run furthest. A gradient-boosted model on those features should
rank candidates by forward Max Favorable Excursion (MFE) with positive IC. The `log_space`
target compresses MFE tails so the model optimizes rank, not a few outsized winners.

Regime (M03) is included deliberately: ablation shows it's a **net-positive mid-pack
contributor** (−0.22 Sharpe if dropped) — a *coincident state descriptor* ("given this regime,
which names work"), not a drawdown predictor. Good state descriptor ≠ good future predictor;
that's why M03 helps M01 selection but fails standalone as market timing.

## 3. Specification

| Aspect | Value (champion `m01_prototype`) |
| :--- | :--- |
| **Algorithm** | XGBoost regressor |
| **Target** | `log_space` = `sign(MFE) · ln(1 + |MFE|)` (log-compressed Max Favorable Excursion) |
| **Feature set** | `fs_m01_prototype` — momentum/RS, volume, alphas, fundamentals, categoricals, 7× `m03_*` regime cols |
| **Universe** | Sparse — SEPA trade candidates only (event-driven, not the dense panel) |
| **Key hyperparams** | `n_estimators=300`, `learning_rate=0.03`, `max_depth=4`, `subsample=0.8`, `colsample_bytree=0.8`, `reg_alpha=5.0`, `reg_lambda=3.0` |
| **Preprocessing** | log-transform any `log_*` feature via `sign(x)·log(1+|x|)`; winsorize outliers |
| **Feature groups (ablation ΔSharpe)** | Core_Volume −0.57 · Fundamentals −0.51 · Momentum_RS −0.41 · MAs −0.26 · Categoricals −0.24 · **M03 −0.22** · Fast_Alphas −0.12 · Oscillators 0.00 · Volatility_Ranges **+0.13** |

**Two-model system:** `m01_prototype` = SELECTION, `m01_rank` = TIMING. They are complementary
— do **not** test `m01_rank` as a standalone picker.

**Macro-strip finding (sprint 13):** macro context is *essential* for the binary "Home Run"
prediction but *redundant* for the standard 4-class multi-class prediction.

## 5. Performance

Honest backtest (bar-by-bar mark-to-market, shared capital pool — the earlier vectorized curve
was inflated and has been retired):

| Measure | Value | Notes |
| :--- | :--- | :--- |
| Arena Sharpe (common window 2025-10→2026-05) | m01_binary **1.997** ≈ m01_prototype **1.992** | dead heat; prototype highest total return (54%) |
| m01_no_macro (4-class) | **0.33** | worst variant |
| **Walk-forward aggregate OOS Sharpe** | **0.84** | 3 folds, 2yr train / 1yr test, re-tuned per fold — **the honest steady-state** |
| WF ann return / maxDD | 79% / −33% | fold-1 (2024) whiffed (IS 1.64 → OOS 0.08) = regime break |

- Arena Sharpes (~2.0) are **regime-flattered** by a short strong-tape window — trust the WF 0.84.
- **Caveat:** prototype in the arena is macro-inclusive; the `m01_*` no-macro variants aren't →
  "prototype vs binary" carries a macro-vs-no-macro difference. Noted in the arena report.
- Ablation artifact: `models/m01_binary/v1/evaluation/full_eval/ablation/ablation_summary.json`.

## 6. Version History / Journey

- **Sprints 1–12 — the selection ranker matured.** Converged on the `log_space` MFE target
  (IC 0.338 winner) and the `fs_m01_prototype` feature set; `m01_prototype_2003_2026/v1` became
  the production scorer feeding `daily_predictions`.
- **Two-model split — prototype (SELECTION) + rank (TIMING).** `m01_rank` added as the timing
  partner; the two are evaluated together, never rank alone.
- **Sprint 13 — strip-M03 experiment.** Built `fs_m01_no_macro`, retrained `m01_no_macro` and
  `m01_binary_no_macro`. **Result:** macro essential for binary, redundant for 4-class → keep M03
  in the champion; strip only where it's provably free.
- **Sprint 13 — honest backtest + arena.** Rebuilt the equity curve (bar-by-bar MtM), ran the
  all-variant arena and walk-forward gate. m01_binary ≈ m01_prototype; single-split optimizer
  overfits (IS 1.22 → OOS −0.17), WF is the honest gate (0.84). See
  [2026-07-02_backtest_arena_session.md](../session_logs/sprint_13/2026-07-02_backtest_arena_session.md).

## 7. Usage

```bash
# Train (scan candidates → enrich features → train regressor → report)
.venv/Scripts/python.exe model_runner.py m01 --steps scan features train
# Full automated workflow (load → EDA → feature-select → train → report)
.venv/Scripts/python.exe model_runner.py workflow --start 2003-01-01 --end 2026-01-01
```

- **Prod scoring path:** M01 scores are materialized nightly into `daily_predictions` (never
  scored live in the dashboard). Prototype is scored via **`daily_predictions` injection** in the
  backtest because it lacks a frozen categorical vocab.
- **Backtest:** `scripts/run_backtest.py --model m01_prototype` (canonical path `score_from_t3()`).
  Only variants with `categorical_mapping.json` load directly (`m01_binary`, `*_no_macro`);
  `m01_prototype` **hard-fails** the scorer and must go through `daily_predictions`.

## 8. Roadmap / Open Questions

1. **Prune dead weight** — ablation says dropping `Volatility_Ranges` *improves* Sharpe (+0.13)
   and `Technical_Oscillators` contributes zero. Candidate feature cuts.
2. **Regenerate prototype categorical mapping** (Option A) or backfill its `daily_predictions`
   (Option B) to widen the prototype backtest comparison window beyond 2025-10 → 2026-06.
3. **VIX sizing through the WF gate** — does OOS Sharpe (0.84) improve with VIX exposure layered
   on? (Sizing is a separate lever from selection — see [regime_model.md](regime_model.md).)
4. **Retirement condition** — if live forward IC decays below the SEPA scanner's own base-rate
   edge over a quarter, the ranking hypothesis is falsified for the live regime.
