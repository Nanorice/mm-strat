## Context/Memory
- **KS threshold insight**: The default 0.15 threshold is industry-standard but strict for this dataset. Top features like `Price_vs_SMA_200` (KS=0.145) barely miss the cutoff. Using 0.10 threshold passed 6 features with IC=0.323.
- **Architecture decision**: Workflow wraps existing `M01Trainer` via composition (not inheritance) to avoid touching working code
- **Feature screening uses Q1 vs Q4 quartiles**: Measures how well features separate bottom 25% returns from top 25%
- **Log-space target confirmed**: Workflow defaults to `log_space` target (IC=0.338 winner from ablation study)

## Usage Quick Reference
```bash
# Full workflow
python model_runner.py workflow --start 2018-01-01 --end 2023-12-31

# EDA only (test new features)
python model_runner.py workflow --steps load eda select --ks-threshold 0.10

# With tuning
python model_runner.py workflow --tune

# Skip auto-selection
python model_runner.py workflow --no-auto-select
```
