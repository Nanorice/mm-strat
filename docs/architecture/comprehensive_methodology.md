## 1. Abstract

An end-to-end pipeline for ingesting, processing, and systematically trading equity
"super performers" per Mark Minervini's SEPA (Specific Entry Point Analysis)
methodology. The nightly pipeline ingests raw data (T1), maintains an investable
universe (screener membership), computes broad technical features (T2), scores the
market regime (M03), maintains SEPA trade sessions (`sepa_watchlist`), computes a
dense feature panel for the SEPA universe (T3), refreshes SQL serving views and the
training cache, scores candidates with the production model into
`daily_predictions`, and publishes a slim replica DB to the remote dashboard.

The production ML model is **`m01_binary`** — a binary XGBoost classifier of the
home-run outcome (MFE > 30% over the SEPA holding period, label
`mfe_binary_homerun_v1`, base rate ~14.5%). It is a **selection** filter used to
gate and rank breakout candidates; it is not a timing or sizing model. The earlier
4-class MFE prototype is archived (binary won the honest-Sharpe start-date cone,
2026-07-15).

Historical validation runs on two engines: a BackTrader event-loop simulation
(fidelity; promotion authority) and a vectorized pandas engine (fast ranking,
optimistic by construction). Live execution does not exist; the champion strategy is
followed as a paper **forward shadow book**. Real fills, entered by hand, live in a
separate append-only portfolio book.

**Epistemic spine** (see glossary §2): a result is a *label* win (C1), an *OOS* win
(C2), or an *exit-P&L* win (C3) — **label lift ≠ trade edge**, and single-run P&L is
one draw: promotion verdicts come from start-date **cones**.

## 3. Data engineering & universe

Sources: **yfinance** (prices, shares, benchmarks), **FMP** (fundamentals backfill,
universe discovery, profiles), **SEC EDGAR** (authoritative filing dates, CIK map,
XBRL deep-history fundamentals, instrument classification), **FRED** (macro),
CNN/AAII (sentiment). Engine-by-engine reference: [engines.md](../modules/engines.md).

- **Universe** = `screener_membership` event log (Phase 2): criteria v2
  (close ≥ $5, 20d avg volume ≥ 100K, market cap ≥ $150M, effective 2020-01-01),
  126-consecutive-day grace period before exit events; point-in-time replayable via
  `screener_criteria_versions`.
- **Survivorship**: delisted tickers are deactivated (`is_active=FALSE`), never
  deleted; non-tradeables (warrants/units/rights/SPACs) are purged and blacklisted;
  renames are patched across all tables. CLIs and JSONL audit trails:
  [manual_for_me.md](manual_for_me.md).
- **Quality gates**: engine-level clamps at write time; Phase 1.5 same-run coverage
  retry; Phase 1.6 plausibility ceilings that withhold the R2 publish; Phase 8
  audit battery (5 audits → `data/audit_reports/*.json`).

## 4. Feature engineering

Two tiers ([feature_pipeline.md](../modules/feature_pipeline.md)):

- **T2** (`t2_screener_features`, 68 cols): full universe daily. SQL pass (SMAs, RS
  line/rating, ranges, ATR/NATR, VCP, `trend_ok`/`breakout_ok`), Python passes
  (9 XS alphas — valid only on the full cross-section; 5 EMAs), SQL rank pass
  (7 universe/sector/industry rank columns).
- **T3** (`t3_sepa_features`, 144 cols): every ticker that has *ever* opened a SEPA
  session, full history. Carries all T2 columns + windowed momentum/volume/velocity
  features, 19 `*_pct_chg` deltas, 9 TS alphas, M03 columns. Fundamentals are
  **not** stored — joined at query time.

**The SEPA gate** `trend_ok ∧ breakout_ok` (exact SQL in glossary §2) is a
**population definition, not an optimization** — scoring un-gated rows inflates the
pool with off-setup days (8.9M → 122K rows). The same scores under a different gate
are a different study.

## 5. Market regime (M03)

`M03RegimeCalculator`: three pillars — Trend 40% (SPY vs SMA200), Liquidity 30%
(Fed net-liquidity slope, release-lag-aware), Risk 30% (VIX + HY spread) → 0–100
score + category (thresholds 80/60/40/20). Output `t2_regime_scores`, joined into
T3 and used by backtest regime gating. A separate 5-factor model writes
`t2_risk_scores`. Reference: [regime_m03.md](../modules/regime_m03.md).

**Falsification record**: M03 does not time entries; SPY>200d is the validated
deploy gate and `stress_ew_vix` the stress axis; multivariate composites failed to
beat SPY-200d; the regime-indicator lever is retired after 5 falsifications. The
regime programme's live goal is *state expression during periods*, not timing.

## 6. SEPA sessions

`sepa_watchlist` (misnomer — it stores **trade sessions**; glossary §1) is the
single session store and T3 universe gate. Entry = first `trend_ok ∧ breakout_ok`
day with no open session; exit = close < SMA50 ∨ SMA150 ∨ SMA200 (**C1+C2+C6
only** — full-template exits fragment sessions via C9 flicker); **no re-entry
cooldown** (removed 2026-07-18; derive `is_retrigger` at read time where episode
dedup matters). `screener_watchlist` is a display view over it. Details:
[managers.md](../modules/managers.md) §2–3.

Sessions are the **trade grain**: one session = one row in `v_d2_training` /
`d2_training_cache` with MFE/MAE outcomes over the session's holding period.

## 7. Machine learning

Full methodology (gates G0–G7, three currencies, promotion rules):
[model_development_methodology.md](model_development_methodology.md). Model
histories: `docs/model_doc/`. Code map: [evaluation.md](../modules/evaluation.md).

- **Problem** (M01 family): given a SEPA breakout today, estimate the probability
  of a home-run excursion (MFE > 30%) over the SEPA holding period. Selection, not
  timing — scores are horizon-invariant (1d vs 20d correlate 0.92), so M01 gates
  and ranks the pool; it does not time entries.
- **Labels** are registry objects (`src/evaluation/label_registry.py`), invariant
  per model line (never swap outcome columns — enrich data instead). Prod label:
  `mfe_binary_homerun_v1`. The 4-class `mfe_4class_v1` and the fixed-horizon tail
  label `m01a_tail_v1` exist for research (two-clocks warning: fixed horizon RANKS,
  SEPA exit HOLDS).
- **Trainer**: `scripts/train_mfe_classifier.py` — XGBoost, features from
  `model_feature_sets`, native categoricals, chronological splits, anchored
  walk-forward, optional isotonic calibration, LeakageGuard + pretrain audit baked
  in. Artifacts + registration: [model_registry.md](../modules/model_registry.md).
- **Prod model**: `m01_binary` (promoted 2026-07-15). Promotion gate =
  `results.json` blocking gates (code) + start-date **cone** (practice); the model
  card is advisory. Nightly it scores both cohorts into `daily_predictions`
  (**RAW score, not a calibrated probability** — the dashboard renders "Score
  (raw)"); a shadow model scores the same universe for divergence monitoring.
- **Two-model system**: `m01_prototype` = selection, `m01_rank` = timing — never
  evaluate `m01_rank` standalone.
- **Selection economics** (falsified-and-kept results): the model earns its keep at
  the **gate**, not within-pool ranking (top-N clip hurts the median; within-pool
  IC ≈ −0.03); breakout-day pool has no persistent within-day separator — triage by
  sector; RS is the home-run axis on the *population* (watchlist axis only — the RS
  trading rule lost the BackTrader cone).

## 8. Backtesting & live shadowing

Reference: [backtest.md](../modules/backtest.md) +
[backtester_manual.md](backtester_manual.md).

- **Scoring**: `UniverseScorer.score_from_t3()` is the canonical path (parity with
  `daily_predictions` is script- and smoke-test-guarded). `ScoreLookup` indexes the
  scored panel for O(1) per-day candidate queries.
- **Engines**: BackTrader `SEPAHybridV1` (async orders, 3-tranche or trail exits,
  M03 regime gating, earnings overlay) is the **promotion authority**; the
  vectorized engine is a fast, optimistic ranking harness (no cash-block,
  charitable stop fills). The forward step-engine (`forward_engine`) mirrors the
  champion for the live **shadow book** (`shadow_book`/`shadow_action`,
  parity-gated ≥0.85 entry overlap).
- **Strategies** are registry configs (`strategy_registry.py`), not subclasses.
  Champion: `champion_trail_spygate` (2026-07-10) — champion selection ×
  trend-exit-only (no tranche TP, SMA50 exit, 15% stop) + SPY-200d ex-ante deploy
  gate.
- **Verdicts are cones**: the edge is start-time dependent (rolling 12m windows:
  Sharpe −0.88..+2.45, 17/53 negative) — a strategy is judged on the start-date
  sweep (`run_starttime_sweep.py`, `run_cone_gate.py`), and OOS via the
  fixed-config gate (`run_oos_gate.py`). Sizing (macro governor `--sizing`) is a
  drawdown controller, not alpha; book-level brakes (DD circuit-breaker,
  earnings blackout) were tested and **rejected** on the cone.

## 10. Replication guide

From-scratch order (each step is a prerequisite of the next):

```bash
# 0. env
python -m venv .venv && .venv/Scripts/Activate.ps1 && pip install -r requirements.txt

# 1. universe seed (FMP discovery → company_profiles)
python scripts/run_universe_backfill.py --discover-fmp

# 2. T1 backfill (shares after prices)
python scripts/run_universe_backfill.py --backfill-prices --start-date 2000-01-01
python scripts/run_universe_backfill.py --backfill-shares
python scripts/backfill_fundamentals.py --source fmp --overwrite
python scripts/backfill_fundamental_ratios.py
python scripts/backfill_t1_macro.py --start 2000-01-01

# 3. data-quality baseline (resolve FAILs before proceeding)
python tools/run_all_audits.py
python tools/purge_junk_tickers.py --execute
python tools/patch_fundamentals.py --fix filing_date_zero
python tools/patch_fundamentals.py --fix filing_date_stale_historical

# 4-5. universe + T2
python scripts/backfill_screener_membership.py      # ~10s
python scripts/backfill_t2_screener_features.py     # verify: audit_t2_screener_features

# 6. regime
python src/regime_pipeline.py --backfill --start 2000-01-01

# 7. sessions (MUST precede T3 — defines its universe; expect ~39K sessions / ~2.7K tickers)
python scripts/backfill_sepa_watchlist.py

# 8. T3 (~9.4M rows, 1.5-2.5h; progress: logs/t3_backfill_progress.log)
python scripts/backfill_t3_sepa_features.py --restart --from 2001-Q1

# 9. views + cache
python scripts/create_duckdb_views.py
python scripts/refresh_training_cache.py            # verify: --stats

# 10. train + card + promote (binary is the prod label)
python scripts/train_mfe_classifier.py --no-holdout --walk-forward --with-calibration \
    --feature-set fs_m01_prototype --label-id mfe_binary_homerun_v1 --model-name m01_binary
python scripts/build_model_card.py --model m01_binary/<version>
# promotion: ModelRegistry().set_prod('<version_id>')  — blocking gates enforced

# 11. strategy verification (cone, not a single run)
python scripts/run_oos_gate.py --strategy champion_trail_spygate
python scripts/run_starttime_sweep.py --strategy champion_trail_spygate --grid rolling

# 12. steady state
python scripts/run_daily_pipeline.py                # or the Prefect deployment
```
