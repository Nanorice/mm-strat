## 💡 Context/Memory

**The key insight from the gate:** M02 predicts *proximity to the scanner event*, and maximum proximity means the pre-breakout move is already spent — the score peaks where the remaining short-horizon return is smallest. This is a principled failure, not a hyperparameter problem. Sharpening event prediction makes it worse as a return predictor. Exit-engineering (E2/E3 asymmetric exits) on a ~0-mean unconditional edge is overfit bait — explicitly declined.

**Selection-bias trap:** The apparent +4.16%-from-signal vs +0.56%-from-entry head-start return conditions on names that *eventually* entered the watchlist. Unconditionally (the full top-50, including the ~50% that never ignite), excess is +12bps/21d pre-cost. Always measure the unconditional top-N return, not the conditioned-on-future-event version.

**WF fold boundary bug fixed:** Consecutive fold specs share their boundary date; the OOS panel builder uses half-open windows `[test_start, test_end)` to avoid duplicate rows.
