Currently, the dashboard has a **D1 Analysis** page that shows basic trade physics (MAE/MFE, E-Ratio), but we're missing comprehensive **feature-level EDA** and **D2R trajectory analysis**. This document proposes a unified analytics system for all three datasets.

#### 2.3 Quantile (Decile) Plots - **The "Monotonicity" Visual**
**Goal:** Catch non-linear relationships that IC misses

**Bar Chart (10 Deciles):**
- X-Axis: Feature Value (Decile 1 = lowest, Decile 10 = highest)
- Y-Axis: Mean Return % (or Probability of Success)
- Color: Gradient from red (low) to green (high)
- Error bars: ±1 std error

**Pattern Recognition:**
- ✅ **Linear (Good):** Bars steadily increase left→right (e.g., `rs_rating`)
  ```
  D1  D2  D3  D4  D5  D6  D7  D8  D9  D10
  5%  7%  9% 11% 13% 15% 17% 19% 21% 25%  ← Steady climb
  ```

- ✅ **Kinked (Good):** Bars flat, then **spike at D9/D10** (e.g., `alpha011`)
  ```
  D1  D2  D3  D4  D5  D6  D7  D8  D9  D10
  8%  9%  8%  9% 10%  9% 10% 11% 18% 28%  ← Only extremes matter!
  ```
  **Action:** This tells you to use a **threshold rule**, not linear weight.

- ❌ **Inverted U (Bad):** Middle deciles perform best
  ```
  D1  D2  D3  D4  D5  D6  D7  D8  D9  D10
  5%  8% 12% 16% 18% 17% 14% 10%  7%  4%  ← Optimization trap!
  ```
  **Action:** Reject this feature. Non-monotonic = overfitting.

**Interactive Selector:**
- Dropdown: Select feature
- Checkbox: `[Show confidence intervals]`
- Toggle: `[Sort by return vs. Sort by feature value]`

### **Tab 3: D2R Trade Physics** (NEW - "What can we learn from the path?")

**Goal:** Optimize Trade Management (stops, targets, holding periods)

#### 3.1 🎯 **CRITICAL: Drawdown Tolerance Curve** (Stop Loss Optimization)
**Question:** Where should my Stop Loss actually be?

**Analysis:**
Calculate `Max_Drawdown_Before_Peak` for all **Winning Trades** only.

**CDF (Cumulative Distribution Function):**
- X-Axis: Max Drawdown % (e.g., -2%, -4%, -6%, -8%, -10%)
- Y-Axis: % of Winners (cumulative)
- Separate lines: All Winners vs. High-Return Winners (>20%)

**Decision Rules:**
- ✅ **If 95% of winners never drawdown >-6%**, but your stop is -10%:
  - **Action:** Tighten stop to -7%. You're burning 4% of risk capital for no reason.

- ⚠️ **If 40% of winners dip to -9% before ripping**, but your stop is -8%:
  - **Action:** Widen stop to -10%. You're killing 40% of your winners prematurely.

**Interactive Slider:**
- User adjusts stop loss: -5% → -15%
- Dashboard shows:
  - % of winners killed by this stop
  - Average return of killed winners
  - Net impact on portfolio return

**Example Output:**
```
Stop Loss Sensitivity Analysis:
  -5%:  Kills 48% of winners, Net Return: +8.2%  ← Too tight
  -8%:  Kills 15% of winners, Net Return: +12.7% ← Current
  -10%: Kills 5% of winners,  Net Return: +14.3% ← Optimal!
```

#### 3.2 MFE vs. Time - **The "Time Decay" of Alpha**
**Question:** How long should I give a trade to work?

**Scatter Plot:**
- X-Axis: Days in Trade (0-60)
- Y-Axis: Max Favorable Excursion (MFE) %
- Color: Final Outcome (Green = Winner, Red = Loser)
- Size: Final Return %
- Trendlines: Winners vs. Losers

**Pattern Recognition:**
- ✅ **"Power Trend":** Winners hit +20% within 10 days
  ```
  Day 5:  +12%
  Day 10: +22%  ← Most alpha captured early
  Day 20: +26%
  Day 30: +28%  ← Marginal gains after Day 10
  ```
  **Action:** If not up +5% by Day 10 → Exit. Alpha has decayed.

- ⚠️ **"Grinders":** Winners take 40 days to hit +10%
  ```
  Day 10: +2%
  Day 20: +5%
  Day 40: +12%  ← Slow burn
  ```
  **Action:** Need patience. Don't use time stops <40 days.

**Implementation:**
- If your data shows 90% of >20% winners happen in <15 days:
  - **Add Time Stop:** If not up +5% by Day 10 → Sell

#### 3.3 "Giveback" Analysis - **Profit Taking Efficiency**
**Question:** Are we holding winners too long?

**Metric:**
```
Efficiency = (Exit_Price - Entry) / (MFE - Entry)
```
- 100% = Exited exactly at peak (impossible)
- 70-80% = Good (captured most of the move)
- <50% = Poor (gave back >50% of gains)

**Histogram:**
- X-Axis: Efficiency % (0-100%)
- Y-Axis: Trade count
- Color by outcome: Green (profitable exit), Red (gave it all back)
- Reference lines: 50%, 70%, 90%

**Insights:**
- **If avg efficiency = 30%**: You only capture 30% of peak move
  - **Action:** Trailing stops too loose. Use tighter profit-taking (e.g., sell half at 2R).

- **If avg efficiency = 85%**: Capturing most of the move
  - **Action:** Strategy is working well.

**Enhanced View:**
- Segment by holding period:
  - <10 days: 45% efficiency (exiting too early)
  - 10-20 days: 65% efficiency (good)
  - >20 days: 25% efficiency (holding too long, giveback)

#### 3.4 Trajectory Clustering (Advanced)
**Goal:** Group trades by price pattern

**Method:**
1. Extract price trajectory for each trade (day 0 to exit)
2. Normalize: `(price - entry_price) / entry_price * 100`
3. Cluster using DTW (Dynamic Time Warping) or K-Means
4. Visualize: Mean trajectory per cluster

**Visualization:**
```python
# Example: 4 clusters
clusters = {
    "Power Trend": [...],    # Steady uptrend, high MFE
    "Spike & Fade": [...],   # Quick spike, then decay
    "Grind Down": [...],     # Slow decline to stop
    "Volatile Chop": [...],  # Wide swings, no direction
}

# Plot mean ± std for each cluster
for name, trades in clusters.items():
    fig.add_trace(go.Scatter(y=mean_trajectory, name=name))
```

**Predictability Analysis:**
