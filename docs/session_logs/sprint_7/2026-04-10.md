## ЁЯТб Context/Memory
- **v0.1 frozen as-is**: The non-negotiable constraint was that registering M01_baseline must not alter the model. `atr_delta` is deliberately registered in the feature set even though it was absent from the trained artifact тАФ the feature set records *intent* (what FEATURE_GROUPS declared), not post-hoc filtering.
- **Catalog belongs in ModelRegistry, not ViewManager**: ViewManager manages SQL views; feature catalog is model metadata. Mixing them would violate layer boundaries.
- **`metadata.json` casing artifact**: `valid_features` in metadata has TitleCase for cross-sectional features (`RS_Sector_Rank`) because DuckDB preserves casing in `UPDATE ... SET RS_Sector_Rank = ...` and this flows through to DataFrame columns. `FEATURE_GROUPS` (the training script) has lowercase. The catalog uses lowercase as canonical. Test was updated to compare against `FEATURE_GROUPS`, not metadata.
- **Log transforms**: 62 lines deleted from view. `log_volume_velocity` was also present in old feature lists (`M01_3BAR_VELOCITY_ONLY`) тАФ it computed `np.log(Volume).diff(2)`, not a view-level transform. Retired in catalog under `version_retired = 'v3.2'`.
- **INSERT OR IGNORE was insufficient for re-seeding**: `populate_feature_catalog.py` now DELETEs existing entries before re-inserting to allow safe re-runs after name corrections.
