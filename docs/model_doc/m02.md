> Model-lifecycle doc: problem, theory, variants, journey, roadmap — the source of
> truth for *why the model is the shape it is* and *how to run it* (§3, §7).
>

## 0. TL;DR / Status

- **Status:** `PROTOTYPE` — validated out-of-sample, **not yet wired into prod** (scanner/dashboard).
- **What it does:** ranks the full ticker universe each day by *how close each name is to
  igniting* — its next SEPA-watchlist breakout. A concentrated shortlist ("watch these ~50").
- **Champion:** `m02_breakout` (XGBoost regressor, target = `breakout_proximity`).
  Latest run `models/m02_breakout/20260629_203134/`.
- **Headline metric:** **Precision@50 ≈ 50%** out-of-sample (2021–2026, 5 anchored WF folds)
  vs a **13.66% base rate** → **~3.6× edge**. Rank IC **+0.33 to +0.39** (mean +0.37).

## 1. Problem & Purpose

M01 ranks candidates *once they've already surfaced* as SEPA trade setups (sparse, event-driven).
Nothing in the pipeline answers the earlier question: **"which names in the whole universe are
about to ignite?"** — a pre-breakout early-warning ranker over *all* tickers, every day.

That's M02's single job: turn the sparse, binary "entered the watchlist" event into a dense,
trackable gradient so we can watch the ~50 names most likely to break out next, before they trip
M01's scanner.

## 2. Theory / Hypothesis

A structural breakout (VCP-style base completion, supply/demand tightening) leaves a
*measurable pre-signature* in price/volume velocity and momentum in the days–weeks before the
event. If that signature exists, a model trained on the dense population should rank names by
time-to-ignition with better-than-random precision — **even though the ignition event itself is
rare (~14% within 60d).** The 50% Precision@50 result (§5) is the evidence that the signature is real.

## 3. Specification

| Aspect | Value |
| :--- | :--- |
| **Algorithm** | XGBoost regressor, `reg:squarederror`, `hist` tree method |
| **Target** | `breakout_proximity = exp(-k · days_to_next_watchlist_entry)`, k=0.1, 0 beyond 60d horizon |
| **Feature set** | `fs_m01_prototype` (dense candidate-population features) via `t3_training_cache` |
| **Universe** | Dense — all tickers, all days (~9.3M rows, 2021–2026 test span) |
| **Key hyperparams** | `max_depth=6`, `eta=0.05`, `subsample=0.8`, `colsample_bytree=0.8`, `min_child_weight=20`, `num_boost_round=300` |
| **CV geometry** | Anchored walk-forward, 1Y step, **60-day embargo** (= horizon) to block lookahead |

**Target construction** (`scripts/build_breakout_targets.py`): join `price_data` against
`sepa_watchlist`, compute calendar days to each ticker's *next* watchlist entry, apply the
exponential decay. Ignites tomorrow → ~1.0; in 30d → ~0.05; no ignition in 60d → exactly 0.0.

## 7. Usage

```bash
# Build dense proximity targets (joins price_data × sepa_watchlist → m02_breakout_targets)
.venv/Scripts/python.exe scripts/build_breakout_targets.py

# Train + anchored-WF eval (checkpointed per fold; --smoke for a 5-ticker path test)
.venv/Scripts/python.exe scripts/train_breakout_model.py --smoke        # smoke first
.venv/Scripts/python.exe scripts/train_breakout_model.py                 # full run
```

- Output: `models/m02_breakout/<run_tag>/` — per-fold `*_model.json`, `fold_NN.json`
  checkpoints, `summary.json`.
- **Not yet in prod.** To go live it must be wired into `daily_scanner.py` and surfaced in the
  dashboard for forward, out-of-sample monitoring (see §8).

## 8. Roadmap / Open Questions

1. **Target stratification** — restrict positives to *quality* watchlist entries
   (`breakout_ok = TRUE AND trend_ok = TRUE`) rather than every entry.
2. **Horizon / decay tuning** — sweep k and the 60d horizon for the sharpest predictive gradient.
3. **Prod integration** — wire `m02_breakout` into `daily_scanner.py`; materialize predictions
   into the dashboard for live forward validation. **This is the gate to leaving PROTOTYPE.**
4. **Retirement condition** — if live forward P@50 collapses toward the 13.66% base rate over a
   quarter, the pre-signature hypothesis (§2) is falsified for the live regime → retire.
