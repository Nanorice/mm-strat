## ⏭️ Next Steps: Phase 2 (Velocity Feature Engineering)

### Goal
Add 5 velocity-specific features to M01_3BAR_FEATURES to improve model's ability to distinguish igniters from drifters.

### Hypothesis
Features measuring **acceleration** and **urgency** will predict TP vs Time better than quality metrics (which predict M01's "will it go up eventually?" question).

### Priority Features

**1. `rs_velocity`** - How fast is RS accelerating?
```python
rs_velocity = (RS[t] - RS[t-5]) / 5  # 5-day slope
```
- **Why:** Igniters show accelerating relative strength
- **Drifters:** RS stays flat or decelerates

**2. `volume_acceleration`** - Is buying pressure increasing exponentially?
```python
vol_accel = (vol[t] - vol[t-1]) - (vol[t-1] - vol[t-2])
```
- **Why:** Igniters show volume surges (accumulation)
- **Drifters:** Volume stays low or declines

**3. `breakout_momentum`** - How explosively did it break the high?
```python
breakout_strength = (close - high_20d) / ATR
```
- **Why:** Igniters break with conviction (3-5 ATR move)
- **Drifters:** Weak breakouts (< 1 ATR)

**4. `consolidation_duration`** - How long was the coil?
```python
days_in_vcp = count(range_daily < 0.5 × ATR)
```
- **Why:** Longer consolidations = more energy stored
- **Drifters:** No clear consolidation pattern

**5. `price_momentum_curve`** - Is price acceleration increasing?
```python
price_accel = (close[t] - close[t-1]) - (close[t-1] - close[t-2])
```
- **Why:** Igniters show parabolic acceleration
- **Drifters:** Linear or decelerating

## 💡 Key Learnings & Design Decisions

### 1. Ignition Score > Expectancy for Meta-Labeling
- **Traditional:** Optimize barriers for max expectancy (absolute returns)
- **Ignition Engine:** Optimize for max separation (discriminative power)
- **Rationale:** Model needs strong signal to learn from, not high returns on barriers
- **Result:** Ignition score = 2.17 (excellent), Expectancy = 0.47% (acceptable)

### 2. Low TP Rate is Acceptable
- **Previous Concern:** TP rate dropped from 11.4% to 5.4%
- **New Understanding:** This is the **signal** - most SEPA entries aren't igniters
- **Implication:** Model should learn to identify the rare 5.4% that DO ignite
- **Analogy:** Metal detector set to "gold only" - ignores iron, finds treasure

### 3. Shorter Horizons Catch Drifters Earlier
- **Previous:** 60-day max → 54% time-outs (many false negatives)
- **Optimized:** 30-day max → 33% time-outs (cleaner signal)
- **Benefit:** Faster feedback loop, less capital wasted on slow movers

### 4. Tighter Stops Increase Capital Velocity
- **Previous:** 2.0× ATR stops → avg 14.5 days holding
- **Optimized:** 1.0× ATR stops → avg 10.5 days holding
- **Benefit:** 28% faster capital turnover = more trades per year

### 5. Phase 2 is Critical for Success
- **Current Model:** Uses M01 features (quality metrics)
- **Problem:** Quality ≠ Velocity (direction ≠ speed)
- **Solution:** Add velocity features (acceleration, urgency)
- **Expected:** +0.5-1.5% selection edge improvement
