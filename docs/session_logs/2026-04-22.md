### Approximate equity curve
The `equity_curve()` method is approximate — it adds PnL dollar amounts at exit dates without tracking intraday portfolio value or position overlap. For parameter sweeps it's fine; for precise Sharpe it may diverge from BackTrader results.

### Exit uses `close` not next-open fill
Entry price = close on entry_date (not next-day open). This slightly overstates returns by eliminating overnight gap slippage. BackTrader fills at next-open by default.

## 💡 Context/Memory

- **Why vectorized over BackTrader for iteration**: BackTrader's event loop with 1826 ticker feeds takes minutes per run. The vectorized engine finishes in seconds because the entire exit logic is a merge + cumulative operation — no Python loops over bars.
- **Single entry per ticker**: The engine currently takes only the *first* breakout entry per ticker across the whole backtest window. This is intentional for simplicity but differs from BackTrader (which can re-enter a ticker after an exit). Could add re-entry support by dropping `drop_duplicates` and instead tracking active positions.
- **T3 already has `sma_50`**: No need to recompute SMA in the vectorized engine — but `price_data` is used for exits (not T3) because T3 only covers active SEPA candidates, not the full holding period post-exit. SMA is computed from `price_data` inside `_load_prices()` using rolling().mean().
- **`atr_delta` still missing from scoring**: The `score_from_t3()` scorer logs ~45 NaN-filled features because the M01 model was trained on `d2_training_cache` (which has `atr_delta`) but T3 doesn't store it. This is a known limitation — not fixed this session.
