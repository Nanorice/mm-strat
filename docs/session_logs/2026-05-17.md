## 💡 Context/Memory
- **Critical lineage correction**: The plan claimed `v_d2_features` is the dense ~9.4M-row daily table. **This is wrong.** `v_d1_candidates` Step 4 keeps "only entry_date row (one row per trade)", so `v_d2_features` is **sparse ~38K rows** (identical to `v_d2_training` minus outcome columns). The actual dense daily table is `t3_sepa_features` (9,298,701 rows). `dense` mode was corrected at runtime to source `t3_sepa_features`.
- **Bad-ticker scale bug in notebook**: `scores_eda.ipynb`'s `detect_bad_tickers` uses bare `500`/`1000` thresholds but `return_*` columns are fractional (1.0 = 100%), so the notebook's checks almost never fire — LIF/CUE were removed via hard-coded list. The ported function uses scale-corrected defaults (5.0 = 500%, 10.0 = 1000%), which correctly detect LIF, CUE, and 6 others.
- **`target_class` in feature list bug**: After `derive_target_class()` adds a `target_class` column, it would appear in IC/MI as a perfect predictor unless explicitly excluded. Fixed by adding `TARGET_COLS = {"target_class", "target_label"}` to the exclusion set in `_select_feature_cols`.
- **`cached_at` leaks into features**: `d2_training_cache` adds a `cached_at` TIMESTAMP column not present in `v_d2_training`. Fixed by adding `"cached_at"` to `METADATA_COLS` in `pretrain_report.py`.
- **`np.isinf` on pandas nullable types**: `to_numpy()` fails with `TypeError` on nullable integer/boolean columns. Fixed in `infinite_audit` by using `to_numpy(dtype="float64", na_value=np.nan)`.
- **IC on categoricals**: `sector`/`industry` are object dtype; `scipy.stats.spearmanr` errors on strings. Fixed by adding `pd.api.types.is_numeric_dtype` guard in `compute_ic` — categoricals participate only in MI (where they're label-encoded).
