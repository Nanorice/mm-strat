## Next Steps

### Priority 1: Universe Snapshot Dataset (Architecture Change)
**Problem**: D2 ranks stocks only against SEPA candidates that passed screening that day. When only 3-5 stocks from a sector pass SEPA, `RS_Sector_Rank` is nearly meaningless.

**Solution**: Create a **daily universe snapshot** dataset:
1. Define the "liquid universe" (e.g., all stocks meeting minimum volume/price criteria)
2. For each trading day, calculate RS and other features for ALL tickers in the universe
3. Store as `data/universe_snapshots/` with:
   - Historical range: `universe_YYYYMMDD_YYYYMMDD.parquet` (for training)
   - Latest day only: `universe_latest.parquet` (for daily scanner efficiency)
4. Cross-sectional ranks should be calculated against this full universe
5. D2 pipeline merges SEPA candidates with pre-computed universe ranks
6. Daily scanner uses same universe for consistent ranking

**Data Flow Change**:
```
Current:  SEPA candidates -> compute features -> rank among candidates
Proposed: Full universe -> compute features -> rank all -> SEPA filters -> merge ranks
```
